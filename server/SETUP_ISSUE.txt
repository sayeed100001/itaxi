⚠️ STEP 3: START MYSQL DATABASE (REQUIRED)
================================================================================

You need MySQL running before migrations can work.

OPTION A: Install MySQL Server
-------------------------------
1. Download MySQL: https://dev.mysql.com/downloads/mysql/
2. Install and start MySQL service
3. Create database: CREATE DATABASE itaxi;
4. Update server/.env with your MySQL credentials:
   DATABASE_URL="mysql://root:YOUR_PASSWORD@localhost:3306/itaxi"

OPTION B: Use XAMPP (Easier)
-----------------------------
1. Download XAMPP: https://www.apachefriends.org/
2. Install XAMPP
3. Start MySQL from XAMPP Control Panel
4. Open phpMyAdmin (http://localhost/phpmyadmin)
5. Create database named "itaxi"
6. Update server/.env:
   DATABASE_URL="mysql://root:@localhost:3306/itaxi"
   (Note: No password by default in XAMPP)

OPTION C: Use Online MySQL (Quick Test)
----------------------------------------
1. Use free service like db4free.net or remotemysql.com
2. Create database
3. Update DATABASE_URL in server/.env

================================================================================

AFTER MYSQL IS RUNNING:
================================================================================

1. Verify MySQL is running:
   - Windows: Check Services (services.msc) for MySQL
   - Or check XAMPP Control Panel

2. Test connection:
   npx prisma db push

3. Run migrations:
   npx prisma migrate dev --name init

4. Start server:
   npm run dev

5. Server should start on http://localhost:5000

================================================================================

QUICK TEST WITHOUT DATABASE (OPTIONAL)
================================================================================

If you want to test without MySQL first, you can:

1. Comment out database operations temporarily
2. Or use SQLite instead (change schema.prisma):
   datasource db {
     provider = "sqlite"
     url      = "file:./dev.db"
   }
